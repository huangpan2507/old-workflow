# Workflow实例存储方案流程图

## 方案A：WorkflowInstance + WorkflowExecutionTask（推荐）

### 完整数据流

```
┌─────────────────────────────────────────────────────────────────┐
│  用户操作：从模板创建实例并执行                                   │
└─────────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│  1. 选择模板 (template_id=1)                                    │
│     ApprovalWorkflowTemplate                                    │
│     ├─ id: 1                                                    │
│     ├─ name: "简历审核流程"                                      │
│     ├─ workflow_nodes: [{id: "start-1", ...}, ...]            │
│     └─ workflow_edges: [{source: "start-1", target: "app-1"}]  │
└─────────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│  2. 配置覆盖                                                     │
│     app_node_configs = {                                        │
│       "app-1": {                                                │
│         "inputMode": "file",                                    │
│         "dagRunId": "dag_run_123"  // 新上传的文件              │
│       }                                                          │
│     }                                                            │
└─────────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│  3. 创建 WorkflowInstance（实例记录）                            │
│     WorkflowInstance                                            │
│     ├─ id: 100                                                  │
│     ├─ template_id: 1  ←──────────┐                            │
│     ├─ status: "pending"          │                            │
│     ├─ execution_result: null    │                            │
│     └─ created_by: user_123       │                            │
│                                   │                            │
│     ✅ 不存储nodes/edges（从模板加载）                           │
└─────────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│  4. 创建 WorkflowExecutionTask（执行引擎需要）                  │
│     WorkflowExecutionTask                                       │
│     ├─ id: 200                                                  │
│     ├─ task_id: "uuid-abc123"                                   │
│     ├─ instance_id: 100  ←───────────┐                         │
│     ├─ nodes: [...]  // 从模板加载    │                         │
│     ├─ edges: [...]  // 从模板加载     │                         │
│     ├─ app_node_configs: {...}  // 合并配置                     │
│     ├─ status: "pending"            │                         │
│     ├─ node_results: {}             │                         │
│     └─ thread_id: "thread-xyz"      │                         │
│                                     │                         │
│     ✅ 存储完整执行信息（执行引擎需要）                         │
└─────────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│  5. 执行Workflow（LangGraph）                                    │
│     - 使用 nodes, edges, app_node_configs                      │
│     - 实时更新 node_results                                     │
│     - 更新 status                                               │
└─────────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│  6. 执行完成                                                     │
│     WorkflowInstance                                            │
│     ├─ id: 100                                                  │
│     ├─ status: "completed"  ←─── 更新                           │
│     └─ execution_result: {  ←─── 更新                           │
│         "final_output": {...},                                  │
│         "summary": {...}                                       │
│       }                                                          │
│                                                                  │
│     WorkflowExecutionTask                                       │
│     ├─ id: 200                                                  │
│     ├─ status: "completed"  ←─── 更新                          │
│     └─ node_results: {  ←─── 更新                              │
│         "start-1": {...},                                       │
│         "app-1": {...},                                         │
│         "end-1": {...}                                          │
│       }                                                          │
└─────────────────────────────────────────────────────────────────┘
```

### 查询示例

```sql
-- 查询模板的所有实例
SELECT * FROM workflow_instances 
WHERE template_id = 1
ORDER BY create_time DESC;

-- 结果：
-- id=100, status="completed", created_at="2024-01-15 10:30:00"
-- id=101, status="running", created_at="2024-01-15 10:35:00"
-- id=102, status="pending", created_at="2024-01-15 10:40:00"

-- 查询实例的执行详情
SELECT * FROM workflow_execution_tasks 
WHERE instance_id = 100;

-- 结果：
-- id=200, task_id="uuid-abc123", status="completed", node_results={...}
```

## 核心区别总结

| 对比项 | 方案A（WorkflowInstance） | 方案B（WorkflowExecutionTask扩展） |
|--------|---------------------------|-----------------------------------|
| **数据存储** | Instance存元数据，Task存执行细节 | Task存所有数据（冗余） |
| **模板修改影响** | ✅ 不影响已创建实例 | ⚠️ 已存储的nodes/edges可能过时 |
| **查询语义** | ✅ 清晰：查询"模板的实例" | ⚠️ 模糊：需要区分临时执行和实例 |
| **数据一致性** | ✅ 高：实例始终使用创建时的模板 | ⚠️ 低：存储的数据可能过时 |
| **扩展性** | ✅ 好：Instance可以添加更多元数据 | ⚠️ 一般：Task职责混杂 |
| **开发工作量** | ⚠️ 需要维护两个表 | ✅ 只需添加一个字段 |

---

## 推荐：方案A

**核心理由**：
1. **数据一致性**：模板修改不影响已创建实例
2. **语义清晰**：Instance = "模板的实例"，Task = "执行任务"
3. **符合设计原则**：模板定义结构，实例记录执行

**实现要点**：
- WorkflowInstance 存储实例元数据（template_id, status, execution_result）
- WorkflowExecutionTask 存储执行过程（关联instance_id）
- 查询时：`SELECT * FROM workflow_instances WHERE template_id = ?`
