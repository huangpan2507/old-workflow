## 3. 文件内容格式化方案对比

### 方案1：纯文本直接显示（不转换为 Markdown）

**原始文本**：
```
这是第一段内容。
这是第二段内容。

这是第三段内容，包含一些特殊字符：<tag> 和 &符号。
```

**UI 显示**：
```
┌─────────────────────────────────────────────────────────┐
│  File Content                                            │
│  ┌──────────────────────────────────────────────────┐  │
│  │ 这是第一段内容。                                 │  │
│  │ 这是第二段内容。                                 │  │
│  │                                                   │  │
│  │ 这是第三段内容，包含一些特殊字符：<tag> 和 &符号。│  │
│  │                                                   │  │
│  │ (纯文本格式，无格式化)                            │  │
│  └──────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

**特点**：
- 保持原始格式
- 可能包含 HTML 标签（如 `<tag>`）
- 特殊字符可能显示为实体（如 `&` 显示为 `&amp;`）
- 段落间距不明显

---

### 方案2：转换为 Markdown 格式显示

**原始文本**：
```
这是第一段内容。
这是第二段内容。

这是第三段内容，包含一些特殊字符：<tag> 和 &符号。
```

**转换后（Markdown 渲染）**：
```
┌─────────────────────────────────────────────────────────┐
│  File Content                                            │
│  ┌──────────────────────────────────────────────────┐  │
│  │ 这是第一段内容。                                 │  │
│  │                                                   │  │
│  │ 这是第二段内容。                                 │  │
│  │                                                   │  │
│  │ 这是第三段内容，包含一些特殊字符：<tag> 和 &符号。│  │
│  │                                                   │  │
│  │ (Markdown 渲染，段落自动分隔，特殊字符转义)       │  │
│  └──────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

**特点**：
- 段落自动分隔（空行转换为段落间距）
- HTML 标签被转义（`<tag>` 显示为文本）
- 特殊字符正确处理（`&` 显示为 `&`）
- 支持 Markdown 语法（如果文本中包含 Markdown）

---

### 方案3：智能格式化（检测并转换）

**如果文本包含 Markdown 语法**：
```
# 标题
这是内容。

- 列表项1
- 列表项2
```

**UI 显示（Markdown 渲染）**：
```
┌─────────────────────────────────────────────────────────┐
│  File Content                                            │
│  ┌──────────────────────────────────────────────────┐  │
│  │ 标题 (大号粗体)                                  │  │
│  │                                                   │  │
│  │ 这是内容。                                        │  │
│  │                                                   │  │
│  │ • 列表项1                                         │  │
│  │ • 列表项2                                         │  │
│  └──────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

**如果文本是纯文本**：
```
这是第一段。
这是第二段。
```

**UI 显示（转换为 Markdown 段落）**：
```
┌─────────────────────────────────────────────────────────┐
│  File Content                                            │
│  ┌──────────────────────────────────────────────────┐  │
│  │ 这是第一段。                                      │  │
│  │                                                   │  │
│  │ 这是第二段。                                      │  │
│  └──────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

**特点**：
- 自动检测文本格式
- 纯文本自动转换为 Markdown 段落
- 如果包含 Markdown 语法，则渲染为格式化内容

└─────────────────────────────────────────────────────────┘

## 总结

### 推荐方案：
1. **文件元信息布局**：方案C（卡片形式，带图标）- 视觉层次清晰，信息分组合理
2. **文件内容格式化**：方案3（智能格式化）- 自动检测并转换，用户体验最佳
3. **加载状态**：方案C（骨架屏）- 提供视觉连续性，专业体验
4. **空状态**：显示友好的提示信息

### 备选方案：
- 如果追求极简风格：方案D（简洁列表）+ 方案1（纯文本）
- 如果追求信息密度：方案B（列表形式）+ 方案2（Markdown 转换）
